## ⚙️ 演算法

> 演算法（Algorithm）是一組有限的、明確的步驟，用來解決某個特定問題。
> 

### ✅ 一個「好」的演算法應具備以下特性：

| 特性 | 說明 |
| --- | --- |
| 有限性 | 演算法必須在有限步驟內完成 |
| 明確性/確定性 | 每一步都必須清晰可執行 |
| 輸入性 | 至少有 0 或多個輸入 |
| 輸出性 | 至少有一個輸出結果 |
| 可行性 | 每個步驟必須能在有限時間內被執行 |

---

## 📏 演算法效率的評價方式

主要從 **時間複雜度** 與 **空間複雜度** 兩個面向來評估。

### 📐 演算法**時間**複雜度三大表示法：

| 記號 | 中文名稱 | 表示什麼？ | 代表意思 |
| --- | --- | --- | --- |
| **O(f(n))** | 大 O（上界） | 最壞情況 | 最慢會這麼慢（最大耗時） |
| **Ω(f(n))** | 大 Omega（下界） | 最好情況 | 最快也只會這麼快（最小耗時） |
| **Θ(f(n))** | 大 Theta（確界） | 平均情況/精確界限 | 不快不慢，實際就這個等級（平均） |

**漸進式的時間複雜度**

```jsx
O(1) < O(log n) < O(n) < O(n log n) < O(n²) < O(n³) < O(2ⁿ) < O(n!)O(nⁿ)
```

---

### 💾 2. **空間複雜度（Space Complexity）**

表示演算法執行過程中**額外佔用記憶體空間的大小，輔助空間和問題的規模**

- 同樣以大 O 表示
- 包括：
    - 變數所需空間
    - 資料結構所佔空間
    - 遞迴堆疊空間等

## 🧠 三、其他演算法評價指標（進階）

| 指標 | 說明 |
| --- | --- |
| 穩定性 | 資料順序是否保持一致（排序） |
| 適應性 | 是否能根據資料特性提升效率 |
| 最壞情況分析 | 對演算法最慢情況的估計 |
| 平均情況分析 | 一般情況下的效能預期 |
|  |  |

### ✏️考題

```jsx
int sum = 0;
for (int i = 1; i < n; i *= 2) ----         // 外層：i 每次乘以 2
    for (int j = 0; j < i; j++) ---        // 內層：j 從 0 跑到 i-1
        sum++;

```

時間複雜度

- 外層執行log₂n
- 內層執行 i 次
    
    分析i
    
    | 第 k 次外層 | i 的值是 2ᵏ | 內層跑次數是 2ᵏ |
    | --- | --- | --- |
    | 第 0 次 | 1 | 1 |
    | 第 1 次 | 2 | 2 |
    | 第 2 次 | 4 | 4 |
    | 第 3 次 | 8 | 8 |
    | ... | ... | ... |

這是 **等比級數**，總和為：

```
複製程式碼
2⁰ + 2¹ + 2² + ... + 2ᵏ ≈ 2⁽ˡᵒᵍ²ⁿ⁾ × 2 = 2n - 1

```

時間複雜度 = O(n)
